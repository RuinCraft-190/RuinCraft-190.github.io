自建NAS方案规划（i3-12100）

⚠️ 未实践 — 本文仅为方案规划，尚未实际搭建，仅供参考。

背景

搭建 7x24h 家庭 NAS 服务器，运行 Emby/Jellyfin、MC 服务器、文件存储、Docker 等服务。

核心约束：

  • 预算：2000-4000 元（不含硬盘）
  • 功耗：30-50W
  • 已有：14TB HDD + 200GB SSD 可复用

硬件配置清单

项目 配置 参考价
CPU Intel i3-12100（4核8线程，支持 Quick Sync） ¥500
主板 H610 ITX/M-ATX（需有 M.2 + 4+ SATA 口） ¥400-500
内存 16GB DDR4（闲鱼二手 2×8GB） ¥300-500
电源 益衡/全汉 Flex ATX 或 DC-ATX（80Plus） ¥200-350
机箱 乔思伯 N6 或 4 盘位 NAS 机箱 ¥300-650
散热 利民 AXP90-X36 等下压式散热 ¥60
启动盘 已有 200GB SSD 复用 ¥0
数据盘 已有 14TB HDD 复用 ¥0

总计：约 ¥1800-2600

内存说明：2025 年起内存暴涨（DDR4 被称为”电子茅台”），全新 16GB 京东价 850+ 元。推荐闲鱼二手（¥300-500），或先买单条 8GB 后续再加。老台式机为 DDR3，新主板不兼容。

功耗表现

场景 功耗
待机 18-25W
MC 服务器运行 25-35W
Emby 转码时 30-45W
年电费 约 ¥100-180

为什么选 i3-12100

  1. MC 服务器流畅 — 单核性能是 N100 的 2 倍+，20+ 人无压力
  2. Emby 4K 转码无压力 — Intel Quick Sync 硬件加速，UHD730 远强于 N100
  3. 功耗可控 — 待机 18-25W，完全在 30-50W 目标范围内
  4. 预算合理 — 1800-2600 元，还有余量买备用盘
  5. 可复用现有硬件 — 14TB HDD + 200GB SSD

软件方案

操作系统

  • 选择：Debian 12 (Bookworm) + Docker
  • 加装 CasaOS 提供 Web 管理界面（可选，类似群晖 DSM 体验）
  • NAS 功能用 Samba 实现文件共享

服务部署

服务 部署方式 备注
Emby/Jellyfin Docker 利用 Intel Quick Sync 硬件转码
MC 服务器 Docker (itzg/minecraft-server) 分配 4-6GB 内存
文件管理 OpenMediaVault 或 Samba 直出 SMB/NFS 共享
下载器 Docker (qBittorrent/Aria2) PT/BT 下载
其他服务 Docker 按需添加

功耗优化

  • BIOS 关闭不用的设备（串口、多余 USB 控制器等）
  • 启用 C-state（CPU 深度节能）
  • 硬盘设置自动休眠（20 分钟无 IO 则停转）
  • 使用 DC-ATX 电源替代传统 ATX 电源（转换效率更高）

购买清单

  1. Intel i3-12100 盒装/散片 + 散热器
  2. 华擎/昂达 H610 ITX 主板（注意选 DDR4 版本）
  3. 16GB DDR4 内存 — 推荐闲鱼二手（¥300-500），或全新单条 8GB 先用
  4. DC-ATX 电源 或 益衡 Flex ATX 电源
  5. 乔思伯 N6 或类似 NAS 机箱
  6. （可选）另一块 HDD 做数据冗余

搭建步骤

  1. 组装硬件，进 BIOS 设置低功耗选项
  2. 安装 Debian 12 系统
  3. 配置 Samba 共享（NAS 功能）
  4. 安装 Docker + Docker Compose
  5. 部署服务（Emby、MC、qBittorrent 等）
  6. 测试 Emby 4K 转码是否正常走硬件加速
  7. 测试 MC 服务器性能（加入游戏看 TPS）
  8. 用智能插座测量实际功耗
  9. 跑 24h 稳定性测试
